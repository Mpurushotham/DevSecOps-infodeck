DevSecOps Stages

Comprehensive Security Integration Across All Phases

1. Plan

+

What Happens

Requirements gathering, architecture design, and security requirements establishment.

Why It Matters

  • Shift-left principle: Security requirements defined upfront
  • Cost efficiency: Fixing security issues in planning is 100x cheaper
  • Compliance: Meeting regulatory requirements from the start

Security Integration

# threat-model.yaml
application: user-auth-service
assets:
  - user-credentials
  - personal-data
threats:
  - threat: Credential theft
    impact: High
    mitigation: Multi-factor auth

Checklist

2. Code

+

What Happens

Developers write code following security best practices with automated checks.

Typical Risks

  • Hardcoded secrets
  • SQL injection vulnerabilities
  • Insecure dependencies

Security Integration

# pre-commit.sh
#!/bin/bash
# Secret scanning
if git diff --cached --name-only | xargs grep -n "password\|secret"; then
    echo "ERROR: Potential secrets found"
    exit 1
fi
